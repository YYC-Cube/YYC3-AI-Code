## 功能限制 (Functional Limitations)

### 🔴 高影响 (High Impact)

#### F-001: Agent自主执行能力尚未实现
- **描述**: 当前版本无法像Cursor或Claude Code那样让AI自主执行多步骤任务（如编写→测试→修复循环）
- **影响**: 用户需要手动复制粘贴生成的代码到IDE中运行
- **当前行为**: 仅提供代码生成和建议，不直接修改文件系统或运行命令
- **计划修复**: v0.2.0 "Agent觉醒"版本（2026-06-15）
- **临时方案**: 手动复制代码到本地编辑器中使用

#### F-002: MCP (Model Context Protocol) 协议未集成
- **描述**: 不支持MCP标准协议，无法与外部工具（如GitHub/Jira/Docker）原生集成
- **影响**: 无法通过标准化接口扩展YYC³的能力
- **当前行为**: 只能使用内置的6大AI提供商
- **计划修复**: v0.2.0（2026-06-15），作为MCP Server和Client双向支持
- **临时方案**: 通过插件系统（v0.2.0后）或手动集成

#### F-003: 实时协作功能受限
- **描述**: 虽然集成了Yjs CRDT库，但多人实时协同编辑功能未完全实现
- **影响**: 团队成员无法同时编辑同一份设计文档
- **当前行为**: 单用户模式为主，无冲突检测机制
- **计划修复**: v0.2.1（2026-07-01），完善WebSocket同步和光标追踪
- **临时方案**: 使用Git进行版本控制和协作

#### F-004: 插件系统尚未上线
- **描述**: Plugin API虽然已在代码中定义，但缺少实际可用的插件和 marketplace
- **影响**: 无法通过插件扩展功能（如新增AI Provider、自定义模板等）
- **当前行为**: 仅支持内置功能
- **计划修复**: v0.2.0（2026-06-15）发布Plugin SDK + 5个官方插件
- **临时方案**: Fork项目自行修改（MIT许可证允许）

---

### 🟡 中等影响 (Medium Impact)

#### F-005: 代码历史版本管理缺失
- **描述**: 无法查看或回溯之前生成的代码版本
- **影响**: 如果误操作或需要对比不同Prompt的结果，无法恢复
- **当前行为**: 每次生成都会覆盖之前的输出（除非手动保存）
- **计划修复**: v0.1.2（2026-05-15）
- **临时方案**: 手动复制重要代码到本地文件备份

#### F-006: 模板市场内容有限
- **描述**: 目前只有基础的项目模板（React/Vue/Next.js），缺乏行业特定模板
- **影响**: 用户可能需要从零开始描述常见场景（如电商后台、管理系统）
- **当前行为**: 约5个官方基础模板
- **计划修复**: v0.1.4（2026-05-29）扩展至20+模板，v0.3.0开放社区模板提交
- **临时方案**: 使用详细的自然语言描述来弥补模板不足

#### F-007: 导出格式单一
- **描述**: 目前仅支持导出为单独的代码文件，不支持打包为完整项目（如ZIP）
- **影响**: 无法一键生成可运行的完整项目骨架
- **当前行为**: 需要逐个文件复制并手动组织目录结构
- **计划修复**: v0.1.0稳定版（2026-05-01）增加项目打包导出
- **临时方案**: 手动创建目录并逐一保存文件

#### F-008: CLI命令行工具缺失
- **描述**: 没有独立的CLI工具（如 `yyc3 generate component`），只能在Web界面使用
- **影响**: 无法在终端/CI/CD流水线中集成YYC³
- **当前行为**: 仅限浏览器GUI操作
- **计划修复**: v0.2.5（2026-08-15）发布完整CLI工具
- **临时方案**: 使用Web界面的API接口（如果有暴露REST API）

## 性能限制 (Performance Limitations)

### P-001: 大规模代码生成时的内存占用
- **描述**: 当一次性生成超过5000行代码时，内存占用可能超过500MB
- **影响**: 在低配设备（<8GB RAM）上可能出现卡顿
- **当前基线**: ~350MB（正常使用），~600MB（极端情况）
- **优化目标**: v0.2.3（2026-08-01）引入虚拟列表和Web Worker，降至<400MB
- **缓解措施**: 分批次生成（每次<1000行），定期刷新页面

### P-002: 首次加载AI模型列表延迟
- **描述**: 启动时加载6大Provider的完整模型配置需要1-2秒
- **影响**: 用户感知到短暂的"加载中"状态
- **当前基线**: FCP ~1.8s（可接受范围）
- **优化目标**: v0.1.3（2026-05-22）优化至<1.5s
- **缓解措施**: 显示加载动画和进度提示

### P-003: 并发请求限制
- **描述**: 同时只能有1个活跃的代码生成任务
- **影响**: 无法批量生成多个组件或文件
- **当前行为**: 排队机制，第二个请求等待第一个完成
- **计划修复**: v0.2.1（2026-07-01）支持并发任务队列（最多3个并行）
- **缓解措施**: 依次生成所需代码

### P-004: 离线缓存能力有限
- **描述**: 断网后只能使用已缓存的静态资源，无法调用任何AI服务
- **影响**: 完全依赖网络连接（除非使用Ollama本地模型）
- **当前行为**: 无Service Worker，无离线PWA支持
- **计划修复**: v0.3.0（2026-09-15）完整PWA + 离线缓存策略
- **缓解措施**: 部署Ollama本地模型以实现部分离线能力

## 兼容性限制 (Compatibility Limitations)

### C-001: 浏览器支持范围
- **描述**: 仅针对现代浏览器的最新2-3个版本进行测试和优化
- **受支持的浏览器**:
  - ✅ Chrome/Edge: 最新版本（Chromium 120+）
  - ⚠️ Firefox: 最新版本（主要功能可用，部分动画可能异常）
  - ❌ Safari: 未充分测试（已知Flexbox Grid布局问题）
  - ❌ IE11及更早版本: 完全不支持
- **计划修复**: v0.1.5（2026-06-05）完成Safari兼容性，v1.0正式支持主流浏览器
- **缓解措施**: 推荐使用Chrome或Edge最新版

### C-002: 移动端体验不完整
- **描述**: 虽然响应式布局已实现，但触摸手势优化和移动专属UI未完成
- **影响**: 手机上可以使用，但体验不如桌面端流畅
- **具体表现**:
  - 触摸目标尺寸偏小（<44px的部分按钮）
  - 缺少滑动返回手势
  - 虚拟键盘弹出时布局可能错乱
  - 长按菜单未实现
- **计划修复**: v1.1.0（2027-Q2）发布React Native原生App
- **缓解措施**: 平板横屏模式下体验最佳

### C-003: Node.js版本要求
- **描述**: 需要 Node.js >= 18.x（推荐20.x LTS）
- **影响**: 使用旧版本Node.js的用户无法运行开发服务器
- **原因**: 依赖了较新的ES Module和Fetch API特性
- **计划修复**: 不打算向下兼容（跟随Node.js release schedule）
- **缓解措施**: 使用nvm管理多个Node版本

### C-004: 操作系统限制
- **描述**:开发和测试主要在macOS上进行，Linux和Windows可能有边缘case
- **已知问题**:
  - Windows路径分隔符（\ vs /）可能导致文件操作异常
  - Linux某些发行版的字体渲染差异
  - macOS特有的Touch Bar支持（非关键功能）
- **计划修复**: v0.1.5（2026-06-05）增加CI矩阵测试（Ubuntu/Windows/macOS）
- **缓解措施**: 使用Docker容器化开发环境（跨平台一致）

## 安全限制 (Security Limitations)

### S-001: API密钥存储安全性
- **描述**: API密钥默认存储在localStorage中（浏览器端），存在XSS攻击风险
- **风险等级**: 🟡 中等（如果网站被注入恶意脚本）
- **当前保护措施**:
  - CSP策略限制脚本来源
  - 密钥在传输前进行AES-256加密
  - 输入净化防止XSS
- **增强计划**: v0.2.2（2026-07-15）引入端到端加密和审计日志
- **最佳实践**: 定期轮换API密钥，不要在公共计算机上使用

### S-002: 速率限制绕过可能性
- **描述**: 前端速率限制可通过禁用JavaScript或修改客户端代码绕过
- **风险等级**: 🟢 低（因为真正的限流应在服务端实施）
- **当前状态**: 纯前端限流（演示用途）
- **生产建议**: 部署时应配合后端代理服务（如Nginx rate limiting）
- **增强计划**: v0.4.0（2026-11-01）企业版提供完整后端服务

### S-003: 代码生成内容的版权模糊性
- **描述**: AI生成的代码可能无意中包含受版权保护的片段（训练数据泄露）
- **风险等级**: 🟡 中等（法律灰色地带）
- **免责声明**: 用户对生成的内容承担全部责任
- **缓解措施**: 
  - 不要直接复制敏感的商业逻辑
  - 对生成代码进行人工审查后再使用
  - 关注AI版权相关的法律进展

### S-004: Ollama本地模型的网络暴露风险
- **描述**: 如果Ollama配置不当（绑定0.0.0.0），局域网内其他设备可访问
- **风险等级**: 🟡 中等（仅影响本地部署场景）
- **缓解措施**: 
  - 确保Ollama只绑定localhost (127.0.0.1:11434)
  - 使用防火墙规则限制外部访问
  - 定期更新Ollama到最新版本

## 测试覆盖限制 (Testing Coverage Limitations)

### T-001: 整体覆盖率偏低
- **当前数值**: 45.73% statements / 35.43% branches
- **目标阈值**: 60% (vitest.config.ts配置)
- **差距**: -14.27% statements / -24.57% branches
- **根本原因**: 
  - 项目快速发展期，功能迭代快于测试编写
  - 复杂业务逻辑（如8阶段管线）测试难度较高
  - UI组件测试需要更多基础设施搭建
- **提升计划**:
  - v0.1.0-beta.1: 55%+（紧急补全核心Store和组件）
  - v0.1.5: 70%+（全面覆盖）
  - v0.2.0: 75%+（含集成测试）
  - v1.0.0: 90%+（GA准入标准）

### T-002: 组件测试严重不足
- **当前状态**: 93个组件中仅15个有对应测试文件（16%）
- **缺失的关键组件**:
  - DesignerPage（主页面）
  - LeftPanel/CenterPanel/PreviewPanel（三面板）
  - DatabaseManager（数据库管理器）
  - PromptBuilder（提示词构建器）
  - TaskBoardPanel（任务看板）
- **优先级**: P0（正在紧急补全中）

### T-003: E2E测试场景有限
- **当前套件数**: 7个
- **覆盖的主要流程**:
  - ✅ 基础导航和路由
  - ✅ 主题切换
  - ✅ 设置页面交互
  - ⚠️ 代码生成完整流程（部分覆盖）
  - ❌ 错误边界处理
  - ❌ 网络异常恢复
  - ❌ 性能回归测试
- **增强计划**: 每个大版本增加3-5个关键E2E场景

### T-004: JSDoc文档覆盖率低
- **当前估计**: ~50%（目测）
- **目标**: >90%（审查标准）
- **缺失最严重的模块**:
  - services/ 目录下的业务逻辑函数
  - components/designer/ 复杂组件
  - utils/ 工具函数
- **自动化方案**: 已准备 scripts/auto-jsdoc.js 批量生成模板

## 规避方案 (Workarounds)

### 针对 Critical/High 问题

#### BUG-001 (Safari Grid布局)
```css
/* 临时修复：在用户的自定义CSS中添加 */
.panel-container {
  display: flex; /* 替代 grid */
  flex-direction: row;
}
```

#### BUG-002 (内存泄漏)
```javascript
// 临时修复：每小时刷新一次页面
setInterval(() => {
  if (performance.memory.usedJSHeapSize > 500 * 1024 * 1024) {
    window.location.reload()
  }
}, 3600000)
```

#### BUG-003 (中文输入法)
```javascript
// 临时修复：使用compositionstart/end事件
element.addEventListener('compositionstart', () => { isComposing = true })
element.addEventListener('compositionend', () => { isComposing = false })
```

### 通用最佳实践

1. **使用Chrome或Edge最新版**获得最佳体验
2. **定期清理浏览器缓存**避免localStorage相关问题
3. **每隔1-2小时刷新页面**预防内存累积
4. **保存重要生成代码到本地文件**防止意外丢失
5. **使用Ollama本地模型**减少对外部API的依赖和网络延迟
