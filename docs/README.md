## 🎯 命名规范说明

### 标准格式

```
YYC3-{大类编号}-{小类简称}-{主题名称}.md
```

**示例**:

- ✅ `YYC3-审核-功能完整性报告.md`
- ✅ `YYC3-MVP-功能设计.md`
- ✅ `YYC3-运维-监控配置指南.md`

### 编号规则

| 编号 | 大类 | 说明 |
|------|------|------|
| 01 | 审核- | 审核报告、测试结果、交付物 |
| 02 | MVP-/规划- | 产品设计、功能规划、路线图 |
| 03 | 运维-/测试- | 运维操作、监控、测试配置 |
| 04 | 社区- | 文化、品牌、社区建设 |
| 05 | 发布- | 版本管理、Release Notes |
| 06 | 规范- | 开发标准、代码规范、指南 |
| 07 | *(无)* | 历史归档（保留原名） |
| 08 | *(无)* | 设计系统（保留原名） |

## 🔄 维护规范

### 新增文档时

1. **确定分类**: 选择合适的 01~08 主目录
2. **遵循命名**: 使用 `YYC3-{分类}-{主题}.md` 格式
3. **更新索引**: 将新文档添加到本文档对应章节
4. **提交PR**: 遵循 `06-开发规范/YYC3-规范-贡献指南.md`

```
新建 → 活跃使用 → 定期 review → 过时归档(07-历史归档/) → 清理删除
```
